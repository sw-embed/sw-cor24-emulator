Fix GitHub issue cor24-rs#2: --terminal mode drops piped input beyond ~4KB. The root cause is that piped stdin is only partially read (~4,110 bytes) before being consumed by the emulator. Buffer ALL piped stdin before starting execution, or stream it on demand without dropping bytes. The precise cliff is 4,112→4,113 bytes. This blocks sw-cor24-pascal and sw-cor24-basic which pipe source code >4KB through --terminal mode. Secondary: detect and warn on stack overflow (SP exits valid EBR range). See the full issue at https://github.com/sw-embed/cor24-rs/issues/2.
